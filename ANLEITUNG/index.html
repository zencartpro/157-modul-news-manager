Sinn und Zweck dieses Moduls

Mit dieser Erweiterung können via Shopadministration Newsbeiträge verfasst werden, die dann im Shop auf der Startseite und/oder in zwei unterschiedlichen Sideboxen angezeigt werden.
Die Beiträge können optional mit einem Start- und/oder Enddatum versehen werden.
Veröffentlichte Beiträge erscheinen zusätzlich in einem Newsarchiv.
Newsbeiträge sind multilingual und für jeden Beitrag können eigene Metatags hinterlegt werden
Seit Version 3.1.2 muss jeder Newsbeitrag in allen im Shop aktiven Sprachen einen Titel und einen Inhalt haben, sonst wird er generell nicht angezeigt
Seit Version 3.1.2 kann mit Newstypen/Newskategorien gearbeitet werden, aus denen sich dann die Beiträge speisen, z.B.
Typ1 = Termine, Typ2 = Neuerscheinungen, Typ 3 = Veranstaltungen, Typ 4 = Aktionen
In den Admin Profilen können unterschiedliche Admins für das Erstellen nur bestimmter Newskategoriebeiträge berechtigt werden
Seit Version 3.1.2 wird für die Newsbeiträge automatisch auch - falls installiert - das Modul Sitemap XML unterstützt, das die Beiträge dann in die Sitemap aufnimmt.
Newsbeiträge aus älteren Versionen dieses Moduls bleiben bei der Installation erhalten und werden automatisch ins neue Format konvertiert.
Ältere Versionen dieses Moduls haben die Einzelbeiträge auf der Seite more_news und das Newsarchiv auf der Seite news_archive angezeigt.
Diese alten Seiten bleiben erhalten und werden automatisch auf die neu berwendeten Seiten article und all_articles weitergeleitet.
Seit Version 3.2.0 kann im Frontend nach Datum/Titl sortiert und gesucht werden.
Hinweis für Updates von älteren Versionen dieses Moduls:
Gehen Sie genauso vor wie unter INSTALLATION beschrieben. Bestehende Beiträge werden übernommen und in die neue Struktur konvertiert.

Neuinstallation

Voraussetzungen


Dieses Modul ist nur für die deutsche Zen Cart 1.5.7 Version geeignet.

Für den Einbau dieses Moduls sollten Sie über folgende Werkzeuge verfügen:

1) Ein guter Texteditor
Damit ist nicht das in Windows enthaltene Notepad oder Microsoft Word gemeint.
Sie benötigen einen Texteditor, der utf-8 versteht und auch im Format utf-8 ohne BOM abspeichern kann.
Meine Empfehlung: UltraEdit (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist der kostenlose Texteditor Notepad++

2) Ein Tool zum Vergleichen von Dateien
Bei der Installation dieses Moduls ist es erforderlich, dass Sie den Inhalt einiger Ihrer bestehenden Zen Cart Dateien mit dem Inhalt der neuen Moduldateien vergleichen und die Änderungen zusammenführen ("mergen").
Meine Empfehlung: BeyondCompare (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist das kostenlose Programm WinMerge


Empfehlung:

Dieses Modul erst in einem Testsystem einbauen und dort konfigurieren/testen und endgültig an die eigenen Wünsche anpassen. Erst dann in einem Liveshop einsetzen! Sichern Sie unbedingt alle Dateien Ihres Shops per FTP und sichern Sie die Datenbank mit phpMyAdmin oder anderen geeigneten Tools!

WICHTIG
Vor dem Einbau dieser Änderungen:
BACKUP von Shop und Datenbank machen!
Keine Haftung, Verwendung auf eigene Gefahr!
BACKUP gemacht? Ok, dann weiterlesen...

Die Installation erfolgt in folgenden Schritten. Halten Sie diesen Ablauf ein!

1)
Im Ordner NEUE DATEIEN den Ordner DEINADMIN auf den Namen des Adminverzeichnisses umbenennen.
Alle Ordner namens DEINTEMPLATE auf den Namen des im Shop aktiven Templates umbenennen.
Dann den Inhalt in der vorgegebenen Struktur hochladen, es werden dabei keine bestehenden Dateien überschrieben.

2)
Im Ordner GEAENDERTE DATEIEN den Ordner DEINTEMPLATE auf den Namen des im Shop aktiven Templates umbenennen.
Vergleichen Sie nun die tpl_index_default.php mit der aus Ihrem Shop und nehmen die Änderung manuell vor, es wird lediglich ein Bereich eingefügt, in dem dann die News auf der Startseite angezeigt werden können:
<?php
//-bof-news_box_manager-lat9 *** 1 of 1 ***
$mnbf = $template->get_template_dir('tpl_modules_news_box_format.php', DIR_WS_TEMPLATE, $current_page_base, 'templates') . '/tpl_modules_news_box_format.php';
if (file_exists($mnbf)) {
require $mnbf;
}
//-eof-news_box_manager-lat9 *** 1 of 1 ***
?>

3)
In die Zen Cart Administration einloggen und auf irgendeinen Menüpunkt clicken.
Unter Konfiguration ist nun der neue Menüpunkt News Box Manager vorhanden. Hier alles wie gewünscht und dort beschrieben einstellen.
Das Verfassen der Newsbeiträge erfolgt dann mit dem neuen Menüpunkt Tools > News Box Manager (Alle Typen) oder gezielt für ein bestimmtes Thema unter den entsprechenden Menüpunkten
Die beiden Sideboxen können unter Tools > Boxlayout aktiviert werden.

KONFIGURATION

Die Themenbezeichnungen für die 4 Artikeltypen können Sie anpassen in:
includes/languages/german/extra_definitions/news_box_manager_defines.php

Unter Administratoren > Admin Profile können unterschiedliche Admins für das Erstellen nur bestimmter Newskategoriebeiträge berechtigt werden.
So könnten Sie z.B. "Redakteure" haben, die nur Aktuellbeiträge zu Terminen, Sonderangeboten oder was auch immer beisteuern.

Die beiden Sideboxen können was Inhalt und Layout betrifft separat konfiguriert werden in den Grundeinstelungen des Moduls.

Bestehende Newsbeiträge können unter Tools > News Box Manager mit den aus der Artielbearbeitung bekannten Symbolen (M = Move = Verschieben, C = Copy = Kopieren) auch in andere Newskategorien verschoben oder kopiert (dupliziert) werden.
Ein duplizierter Beitrag ist erstmal deaktiviert, genauso wie ein duplizierter Shopartikel.

OPTIONAL
Falls Sie das Modul Sitemap XML installiert haben, laden Sie nun noch die Datei includes/modules/pages/sitemapxml/sitemapxml_boxnews3.php aus dem Ordner OPTIONAL SITEMAP XML ins Shopverzeichnis hoch.
Falls sie dort bereits vorhanden ist, einfach überschrieben.
Falls Sie dort noch eine alte sitemapxml_boxnews.php haben, können Sie diese nun löschen.
Nun können Sie unter Tools > Sitemap XML unter Sitemap Plugins das Plugin sitemapxml_boxnews3.php ankreuzen und speichern, um ab sofort auch die Newsbeiträge in der neuen Struktur in Ihre XML Sitemap zu übernehmen.

DEINSTALLATION

Um das Modul vollständig zu entfernen, gehen Sie bitte wie folgt vor:

1)
Die geänderte tpl_index_default.php wieder mit der aus Ihrem Backup ersetzen.

2)
Alle von diesem Modul neu hinzugefügten Dateien vom Server löschen

3)
Im Ordner UNINSTALL die uninstall.sql mit einem Texteditor öffnen, Inhalt kopieren und ins Eingabefeld unter Tools > SQL Patches installieren einfügen und absenden
